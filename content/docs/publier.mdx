Webgen vous permet de publier votre site sur un sous-domaine dédié en quelques secondes, sans configuration serveur ni hébergement à gérer.

## Le bouton Publier

Dans l'en-tête de l'éditeur, cliquez sur le bouton **Publier**. Une modale s'ouvre pour configurer la publication.

## Choisir un slug

Le slug est l'identifiant unique de votre site dans l'URL. Il doit respecter les règles suivantes :

- Uniquement des **lettres minuscules, chiffres et tirets** (`a-z`, `0-9`, `-`)
- **Entre 3 et 50 caractères**
- Pas de tiret en début ou en fin
- Exemple valide : `nutricoach-app`, `portfolio-2026`, `mon-restaurant`

L'URL finale de votre site sera : `https://[slug].webgen.app/s/[slug]`

## Ce qui est publié

La publication inclut :

- **Toutes les pages** du site avec leur contenu et leurs sections
- Le **thème complet** (7 variables CSS + police)
- Les **polices** chargées via Google Fonts CDN
- Les **métadonnées SEO** : balises Open Graph, Twitter cards, description et URL canonique

## Mise à jour d'un site publié

Pour mettre à jour votre site après des modifications, cliquez à nouveau sur **Publier** et utilisez le **même slug**. La publication existante est écrasée avec le nouveau contenu. L'URL reste identique.

<Callout variant="warning">
Les slugs sont **uniques sur toute la plateforme**. Si le slug que vous souhaitez est déjà utilisé par un autre utilisateur, vous devrez en choisir un autre. Choisissez un slug significatif dès le début — il fait partie de votre URL publique.
</Callout>

## Partager votre site

Une fois publié, l'URL est affichée dans la modale et copiable en un clic. Le site est accessible publiquement sans authentification — partagez-le librement.

## Dépublier

La suppression de la publication se fait depuis le tableau de bord, dans les options de votre projet.
